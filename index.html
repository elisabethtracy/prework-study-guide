HTML

  • The [head] element contains information (metadata) about the webpage, and this information is not visible to the user.
  • The [body] element represents the visible content shown to the user.
  • The [id] attribute is read as a way to identify elements and select them.
  • The [h] elements are used to create headings.
  • The [ul] element represents an unordered list of items.
  • The [ol] element represents an ordered list of items.
  • The [li] element represents the list of items.
  • The declaration [!DOCTYPE ...] tells the browser what document type to expect when it is opened.
  • The [lang=en] element tells the browser the language for our webpage.
  • The [title] element defines the title for a webpage.
  • The [style] element defines the CSS styles associated with a website.
  • The [base] element defines the base URL for a webpage.
  • The [link] element connects an external resource to the HTML document.
  • The [meta] element defines metadata such as the character set (charset, sets our character encoding), description, keywords, author, and viewport (sets the default view for our webpage).
  • The [script] element embeds client-side scripts in the HTML document.
  • The [http-equiv] element sets an HTTP header for content.
  • The [p] element represents a paragraph or block of text.
  • The [img] element contains information about images that are displayed on the webpage.
  • The [src] element is the most important attribute of the [img] element; it defines the location of the image.
  • The [br] element creates a line of empty space, or a line break, between two blocks of content.
  • The [a] element (anchor) creates links to the same webpage or other webpages.
  • The [href] attribute points to the URL for the link within an [a] element.

CSS

  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside an element.
  • A content indicates the area where your content is displayed; size it using properties like width, height, block-size, etc.
  • A border wraps the content and any padding; size it using border and related properties.
  • When using an external style sheet, the file must be linked using a [link] element placed in the HTML's head.
  • A CSS rule has the following anatomy: selector {property: value; property: value;}
  • A selector defines the element or attributes to which the rules, or declarations, will apply; separate multiple selectors by commas.
  • Declarations contain both the CSS property we want to apply and the value of the property.
  • To apply a rule to all the elements visible on the page, use the special wild card selector *.
  • A period preceding a selector designates it as a class selector and applies these CSS property assignments to any HTML elements assigned to the class.

Git

  • git status: checks what branch we are currently on.
  • git checkout -b branch-name: creates a new branch and switches to it.
  • git add -A: adds modifications in the current working branch to the staging area (the -A flag indicates that we want to add all changes).
  • git commit -m "commit contents": commits staged files (the -m flag indicates a commit message that will describe what the commit contains).
  • git pull origin main: used to check if our local branch is in sync with the base branch in GitHub.
  • git push origin branch-name: pushes our changes to the remote branch.

JavaScript

  • Variables are containers that store values; declare a variable with the [var] keyword followed by the name you assign to the variable.
  • JavaScript is case-sensitive.
  • Variable = value gives the previously (or in the same line) declared variable a value.
  • A string value (a sequence of text) is enclosed in single or double quotation marks.
  • A number value does not have quotes around it.
  • A Boolean value (true/false) does not have quotes around it.
  • An array value is a structure that allows you to store multiple values in a single reference; var arrayname = ["string", #, true/false, ...];
  • An array is a single variable that is used to hold a group of data. Arrays are typically used to hold data that is related in some way.
  • An object value can be anything; everything in JavaScript is an object and can be stored as a variable.
  • Comments are snippits of text that can be added along with code which the browser ignores; marked with /* comment */ OR // comment.
  • An operator is a mathematical symbol that produces a result based on two values (or variables).
  • Addition operator (+) adds two numbers together or combines two strings.
  • Subtraction (-), multiplication (*), and division (/) operators function as in basic math.
  • Assignment operator (=) assigns a value to a variable.
  • Strict equality operator (===) performs a test to see if two values are equal and of the same data type; will return a Boolean result.
  • Not (!) and does-not (!==) equal operators return the logically opposite value of what it precedes (i.e., turns true to false and vice versa).
  • Click here for a full list of JavaScript operators and expressions!>
  • Conditionals are code structures used to test if an expression returns true or not; commonly represented as if...else.
  • The expression inside the if() is the test; if the comparison runs true, the first block of code runs.
  • Functions are a way of packaging functionality that you wish to reuse; typically appears like variable = value(argument).
  • Events are code structures that listen for activity in the browser and run code in response.
  • An anonymous function doesn't have a name; an alternative way to write them is called an arrow function, and appears () => instead of function ():.
  • Primitive values refer to all types of values except objects that define immutable values represented directly at the lowest level of the language.
  • All primitive types (except null and undefined) can be tested by the typeof operator.
  • Conceptually, undefined indicates the absense of a value, while null indicates the absence of an object; null is a keyword, but undefined is a normal identifier that happens to be a global property.
  • NaN is a special kind of number value that's typically encountered when the result of an arithmetic operation cannot be expressed as a number; it is also the only value in JS that is not equal to itself.
  • BigInt type values are a numeric primitive in JS that can represent integers with arbitrary magnitude (even beyond JS's safe integer limit).
  • A BigInt is created by appending n to the end of an integer or by calling the BigInt() function.
  • JavaScript strings are immutable. This means that once a string is created, it is not possible to modify it.
  • A Symbol is a unique and immutable primitive value and may be used as the key of an Object property; their purpose is to create unique property keys that are guaranteed not to clash with keys from other code.
  • Objects are the only mutable values in JS and can be seen as a collection of properties; either data or accessor properties which have corresponding attributes.
  • Data properties associate a key with a value and can be described by the attributes value, writable, enumerable, or configurable.
  • Accessor properties associate a key with one of two accessor functions (get and set) to retrieve or store a value and can be described by the attributes get, set, enumerable, or configurable.
  • Objects include dates, indexed collections, keyed collections, structured data, and more.
  • To open Chrome DevTools use Ctrl+Shift+I.
  • Control flow is the order in which a computer executes code in a script; runs from top to bottom.
  • An if statement: if (condition) { // block of code to be executed if condition is truthy }.
  • A value is considered 'falsy' if it is one of the following values: 0, -0, On, "", null, undefined, NaN, or a Boolean data type with the value of false.
  • A piece of data's index is used to access individual data inside an array; the first item in any array has an index of 0.
  • A for loop uses the predictable pattern of indices to perform a task on all the items in an array by allowing a single code block to be executed over and over.
  • A for loop contains 3 statements: var x = 0; determines the starting point for our loop. The condition allows the loop to execute 1 time for each item in the array. The final statement is what allows the array to iterate over each item.
  • A for loop: for (var x = 0; x < array.length; x++) { code block }.
  • A function is simply a set of instructions that tells the computer how to perform a certain task; they must be called by name in order to execute.
  • To declare a function use the function keyword and name the function followed by parentheses; add curly brackets {} that contain the code for the task to perform.
