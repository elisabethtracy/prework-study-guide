HTML

  • The [head] element contains information (metadata) about the webpage, and this information is not visible to the user.
  • The [body] element represents the visible content shown to the user.
  • The [id] attribute is read as a way to identify elements and select them.
  • The [h] elements are used to create headings.
  • The [ul] element represents an unordered list of items.
  • The [ol] element represents an ordered list of items.
  • The [li] element represents the list of items.
  • The declaration [!DOCTYPE ...] tells the browser what document type to expect when it is opened.
  • The [lang=en] element tells the browser the language for our webpage.
  • The [title] element defines the title for a webpage.
  • The [style] element defines the CSS styles associated with a website.
  • The [base] element defines the base URL for a webpage.
  • The [link] element connects an external resource to the HTML document.
  • The [meta] element defines metadata such as the character set (charset, sets our character encoding), description, keywords, author, and viewport (sets the default view for our webpage).
  • The [script] element embeds client-side scripts in the HTML document.
  • The [http-equiv] element sets an HTTP header for content.
  • The [p] element represents a paragraph or block of text.
  • The [img] element contains information about images that are displayed on the webpage.
  • The [src] element is the most important attribute of the [img] element; it defines the location of the image.
  • The [br] element creates a line of empty space, or a line break, between two blocks of content.
  • The [a] element (anchor) creates links to the same webpage or other webpages.
  • The [href] attribute points to the URL for the link within an [a] element.

Git

  • git status: checks what branch we are currently on.
  • git checkout -b branch-name: creates a new branch and switches to it.
  • git add -A: adds modifications in the current working branch to the staging area (the -A flag indicates that we want to add all changes).
  • git commit -m "commit contents": commits staged files (the -m flag indicates a commit message that will describe what the commit contains).
  • git pull origin main: used to check if our local branch is in sync with the base branch in GitHub.
  • git push origin branch-name: pushes our changes to the remote branch.
